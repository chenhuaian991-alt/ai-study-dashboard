# CLAUDE.md

## Claude Code 工作方式

Claude Code 是本项目的主要开发助手，负责项目骨架搭建、功能实现、UI 初稿、常规 bug 修复和基础测试。

## 每次开始任务前

先阅读以下文件：

1. AGENTS.md
2. PROJECT_PLAN.md
3. README.md
4. CONTEXT.md，如果存在

阅读后先确认当前阶段、任务范围和下一步计划。

## 执行规则

- 每次只完成一个明确功能
- 大改动前先给出简短计划
- 不一次性生成过多文件
- 不主动扩大需求
- 不在用户未确认时引入后端、数据库、登录、云同步或 AI API
- 不把实验性功能混进 MVP
- 不做与当前任务无关的重构

## 修改后必须汇报

每次完成修改后说明：

- 改了哪些文件
- 实现了什么
- 如何运行或验证
- 还有哪些未完成

## 和 Codex 的分工

Claude Code 负责：

- 主要功能开发
- 项目骨架搭建
- UI 初稿
- 常规 bug 修复
- 基础测试

Codex 负责：

- 文档审查
- 代码审查
- 架构检查
- 测试建议
- 复杂问题的第二意见

## 当前阶段要求

当前处于阶段 0：项目启动与规则建立。

如果用户还没有明确要求初始化项目，请不要创建 Vite 项目、不要安装依赖、不要创建 src 目录。

阶段 1 从用户明确确认“开始初始化项目”后开始，只做 Vite + React + TypeScript 空壳、基础目录结构和核心类型定义。

## 建议给用户的下一步

确认 README.md、PROJECT_PLAN.md、AGENTS.md 和 CLAUDE.md 是否符合预期。

确认后，再进入阶段 1：初始化 Vite + React + TypeScript 项目。
