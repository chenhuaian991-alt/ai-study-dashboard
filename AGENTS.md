# AGENTS.md

## 项目背景

这是一个面向大一学生的本地 AI 学习任务管家，用于管理课程任务、AI 编程日志和每日复盘。

## 通用启动指令

每次开始新任务时，先阅读：

1. AGENTS.md
2. PROJECT_PLAN.md
3. README.md
4. CONTEXT.md，如果存在

CONTEXT.md 固定放在项目根目录。读取上下文后继续工作，不要重复已完成的步骤。

## 对话角色确认

每次新对话开始时，如果用户没有明确说明本次对话用途，需要先询问本次对话角色。

可选角色：

- 项目经理对话：长期保留，用于协调项目节奏、决定下一步该开哪类对话、生成或审核各类提示词、维护上下文
- 规划对话：讨论方向、拆分版本、生成给 Claude Code 或 Codex 的提示词，不直接写代码
- 实现对话：根据明确任务修改代码，完成后运行验证
- 审查对话：review 代码、测试结果和风险，优先给问题等级
- 复盘对话：总结进展、更新文档、沉淀经验

如果用户已经明确说明任务，例如“实现 v0.2 首页概览”或“review 当前代码”，则直接按对应角色执行，不需要重复询问。

## AI 分工

- Codex：主要负责项目规划、任务拆分、提示词生成、代码审查、风险判断和阶段复盘
- Claude Code：主要负责根据 Codex 生成的明确提示词写代码，也可以回答用户关于项目的小问题
- 用户：负责确认方向、验收体验、决定是否进入下一阶段

默认流程：

1. 在项目经理对话中判断当前阶段需要规划、实现、审查还是复盘
2. 必要时新开规划对话，确定版本方向和任务拆分
3. 由项目经理对话整理并生成给 Claude Code 的实现提示词
4. Claude Code 在实现对话中完成代码修改和验证
5. Codex 在审查对话中 review 结果并分级问题
6. 只修复重要问题，再回到项目经理对话决定下一步

## 通用开发原则

- 优先完成可用 MVP
- 每次只完成一个明确功能
- 不主动扩大需求
- 不主动引入复杂依赖
- 不做无关重构
- 遵循现有项目结构和命名风格
- 修改前先说明计划
- 修改后说明改了哪些文件、实现了什么、如何验证

## 代码质量要求

- TypeScript 类型要清晰
- 组件职责要单一
- 数据结构要易于理解
- 不把所有逻辑堆进一个大文件
- 用户数据保存和恢复逻辑要谨慎处理
- 对导入 JSON 等容易出错的功能要有错误处理

## 存档指令

当用户说“存档”或“保存上下文”时：

1. 把当前对话的关键信息压缩写入工作区的 CONTEXT.md
2. 必须用覆盖模式写入，不要追加
3. 控制在 20 行以内
4. 使用中文

## 阶段收尾指令

当一个版本或完整任务链路完成后，如果用户说“阶段收尾”或“完成本轮收尾”，需要：

1. 更新 DEVLOG.md，记录本轮完成内容、验证结果和经验
2. 更新 PROJECT_PLAN.md，标记当前阶段状态和下一步建议
3. 覆盖写入 CONTEXT.md，保留 20 行以内的最新上下文
4. 如果用户可见功能发生变化，检查是否需要更新 README.md
5. 不要自动提交 git，除非用户明确要求
