# AI Study Dashboard v0.2 首页概览设计

## 目标

v0.2 增加一个轻量混合版首页概览 Dashboard，让用户打开应用后可以快速了解当前学习状态：任务压力、近期要做的事、最近 AI 编程日志和每日复盘记录。

本版本继续保持本地单页应用，不引入路由、新依赖、后端、数据库或复杂统计图表。

## 非目标

- 不做登录、云同步、AI API、提醒通知。
- 不做复杂图表、趋势分析、日历视图。
- 不做任务拖拽排序、搜索筛选。
- 不改动 v0.1 已有数据结构，除非发现必须修复的数据问题。

## 页面结构

Dashboard 放在现有页面顶部，原有模块继续保留在下方，顺序建议为：

1. 页面标题
2. 首页概览 Dashboard
3. 任务看板
4. AI 编程日志
5. 每日复盘
6. 数据备份

不引入多页面路由，避免扩大复杂度。

## Dashboard 内容

### 顶部统计卡片

显示 4 个小型统计卡片：

- 待做任务数：`status === 'todo'`
- 进行中任务数：`status === 'in-progress'`
- 已完成任务数：`status === 'done'`
- 本周复盘数：日期在本周范围内的每日复盘数量

统计卡片只展示数字和短标签，不做图表。

### 近期任务

显示最多 5 条任务，优先级规则：

1. 进行中任务排在前面。
2. 待做任务按截止日期从近到远排序。
3. 没有截止日期的待做任务排在有截止日期任务后面。
4. 已完成任务不进入近期任务列表。

每条任务展示标题、课程、状态、截止日期和优先级。空状态显示“暂无近期任务”。

### 最近记录

分为两个简短列表：

- 最近 3 条 AI 编程日志，按 `createdAt` 从新到旧。
- 最近 3 条每日复盘，按 `date` 从新到旧。

每条记录只展示标题或日期、状态摘要，不展开完整内容。空状态分别显示“暂无日志记录”和“暂无复盘记录”。

## 组件拆分

建议新增：

- `src/features/dashboard/DashboardOverview.tsx`：Dashboard 容器组件。
- `src/features/dashboard/StatCard.tsx`：统计卡片。
- `src/features/dashboard/RecentTasks.tsx`：近期任务列表。
- `src/features/dashboard/RecentRecords.tsx`：最近日志和复盘列表。

Dashboard 直接读取现有 storage 函数：

- `getTasks`
- `getAiLogs`
- `getDailyReviews`

不新增全局状态管理。现阶段 Dashboard 在页面首次渲染时读取 localStorage 即可；如果后续发现新增数据后顶部不刷新，再通过父组件刷新机制处理。

## 数据与边界

- localStorage 读取失败时继续使用空数组，不阻塞页面。
- 任务截止日期为空时不参与日期排序优先级。
- 日期格式异常的数据放在列表后面，不抛出错误。
- 本周复盘统计以本地时间为准。

## 样式原则

- 延续 v0.1 的普通 CSS 和视觉风格。
- 统计卡片保持紧凑，不做大面积 hero。
- 桌面端可以使用 4 列统计卡片；小屏下改为 2 列或 1 列。
- 不使用新的 UI 库或图标库。

## 验收标准

- 页面顶部出现 Dashboard 概览区。
- 统计卡片数字与当前 localStorage 数据一致。
- 近期任务最多显示 5 条，并符合排序规则。
- 最近 AI 日志最多显示 3 条，按最新优先。
- 最近复盘最多显示 3 条，按日期最新优先。
- 无数据时各区域有清晰空状态。
- 刷新页面后 Dashboard 仍能正确读取已有数据。
- `npm run lint` 通过。
- `npm run build` 通过。
